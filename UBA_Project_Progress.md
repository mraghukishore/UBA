## ✅ Day 2 — Document Ingestion & ChromaDB Indexing (COMPLETE)

### Project folder structure created:
```
C:\ADDS\Project\UBA\
│
├── docs\                        ← 5 utility PDF documents
├── chroma_db\                   ← ChromaDB vector store (auto-created)
├── app\
│   ├── ingest.py                ← Document loader + embedder script
│   └── verify.py                ← ChromaDB verification script
└── venv\                        ← Virtual environment
```

### Input documents placed in `docs\`:
| File | Pages | Content |
|---|---|---|
| `01_tariff_rate_schedule.pdf` | 2 | Electricity slabs, gas rates, Time-of-Use tariff |
| `02_billing_policy.pdf` | 2 | Billing cycle, bill components, late payment, security deposit |
| `03_faq_customer_queries.pdf` | 3 | 15 detailed Q&As — billing, meters, payments, outages |
| `04_payment_options_procedures.pdf` | 2 | Payment channels, due dates, refunds, instalment plans |
| `05_outage_complaint_process.pdf` | 2 | Reporting channels, restoration timelines, escalation |

### What `ingest.py` does:
1. Reads all 5 PDFs using `PyPDFLoader`
2. Splits text into chunks of 500 characters with 50 overlap using `RecursiveCharacterTextSplitter`
3. Converts chunks to vector embeddings using `nomic-embed-text` via Ollama
4. Stores 45 chunks into ChromaDB at `chroma_db\`

### Verification result (`verify.py`):
- ✅ Total chunks in ChromaDB: **45**
- ✅ Sample query *"What is the late payment charge?"* returned 3 correct relevant chunks
- ⚠️ Deprecation warnings noted — to be fixed on Day 3

## ⏳ Day 3 — RAG Pipeline (NEXT SESSION)

### Goal:
By end of Day 3, you can type a billing question in the terminal and get an intelligent answer generated by llama3 from your documents.

### Activities planned:

#### Activity 1 — Fix deprecation warnings (~10 min)
Upgrade to latest LangChain packages:
```bash
pip install -U langchain-ollama
pip install -U langchain-chroma
```
Update imports in `ingest.py` and `verify.py` from:
```python
from langchain_community.embeddings import OllamaEmbeddings
from langchain_community.vectorstores import Chroma
```
to:
```python
from langchain_ollama import OllamaEmbeddings
from langchain_chroma import Chroma
```

#### Activity 2 — Build the RAG pipeline script (~30 min)
Create `app/rag_pipeline.py` that:
- Loads ChromaDB vector store
- Takes a user question
- Retrieves top 3 relevant chunks
- Passes chunks + question to llama3 via Ollama
- Returns a grounded, accurate answer

#### Activity 3 — Test with real billing questions (~15 min)
Test queries to run:
- *"What is the late payment charge after 30 days?"*
- *"How do I set up auto-pay?"*
- *"What are the domestic electricity tariff rates?"*
- *"How do I report a power outage?"*
- *"What happens if my meter is faulty?"*

#### Activity 4 — Review and tune answers (~5 min)
Check if answers are accurate and grounded in the documents. Adjust if needed.
